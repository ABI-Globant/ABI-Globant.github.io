  1. AEM Siteadmin brief introduction

The Adobe Experience Manager (AEM) CMS it’s a content management solution for building websites, mobile apps and forms.  AEM works with credentials and once you login into it you can start working on the management of your site.

There are two sides to each site: Author mode and Publish mode. The Author mode is accessed by the AEM and it’s where all the changes and construction of the site happens. The Publish mode is the part of the site that has been activated for the public to see. It’s the live version of the site.

For the AB-InBev’s sites two CMS environments were set: Stage and Production. Production is the Live site, the actual website while Stage is an environment to test developments before moving them Live.

In Stage you can modify and test your content without affecting the Production environment. In Production you can modify content and publish it for site visitors to see the new content. It is a best practice to first do changes in Stage environment, verify they work fine and are the expected results, and then to replicate those changes in Production.

The functionality of AEM is made available through various specialized consoles. The naming and/or look-and-feel of these is dependent on which of two UIs you are using:  

The Touch-Optimized UI where the main consoles can be accessed from the navigation tab of the rail (dark menu on the left).

The Classic UI where a selection of the main consoles is provided. The icon toolbar at the top allows you to navigate to other consoles. This is the one described in this document.

The following are the respective urls to the Stage and Production environments:

Stage URL: https://author-abi-stage.adobecqms.com/

Production URL: https://author-abi-prod.adobecqms.com/

After you have logged in into the CMS, click on the Sites option from the left menu. Once the option has loaded hover over the same (Sites) option until a monitor icon appears, then click on the monitor. This will take you to the siteadmin on a new browser tab.

There you will see the following icon toolbar at the top:

This document only describes the consoles involved in Anheuser-Busch sites creation.

  1. Websites

Website console lets you create, view and manage websites running on the AEM instance. It lets you create, copy, move and delete website pages, start workflows and activate (publish) pages, as well as edits its properties or open a page for editing.

It has a content tree on the left to navigate through global sites and regional sites until you get to the desired one.

It has a panel on the right with page information such as Title (name of the page), Name (url of the page), Published (if it was activated), Modified, Status, Impressions, Templates.

 

Expand items in the content tree until you get to the desired item and select it. Its information will be loaded in the right part of the console.

Double-clicking on a page on the right, opens that page for edition in a new browser tab.

  1. Digital Assets Manager (DAM)

This console lets you import and manage digital assets such as images, videos, documents and audio files. These assets can then be used by any website running on the same AEM instance.

Here you can review DAM supported formats.

You should select the site you want to add assets for in the content tree on the left. Then click on New in the right panel and browse to look for the image to upload or drag and drop it.

The asset/s will be added in the last page and after that it/them will be available for every site in the AEM instance.

For images, the maximum size allowed is full high definition (1920 by 1080 pixels). If you upload a larger image, AEM will adjust/crop it to this size.

Assets need to be activated in order to be used in a site. However, if you activate a page that uses an asset that was not activated, a dialog box will appear. It will show a checked checkbox, the name of the asset and the action activate. If you click ok, assets and page will be activated.

  1. Campaigns

The console enables you to manage marketing campaigns. It allows you to create, edit and delete campaigns as well as set some properties.

To add tags for your site click the fourth (the one with the megaphone) icon of the top menu of the System Admin UI.

This will take you to the Campaign section of the CQ5. Once there click on the campaign option from the left menu. This will display the different kind of campaigns (folders) created on the site.

To create a New Campaign start by creating the brand you want it for (if it does not exist) by clicking on the New button on the right side of the page and the next window will appear:

 

The Title field is to type in the title of the element to be created, in this case the name of the brand (ie. budweiser).

The Name field is to type in the name of the brand, is preferable to use the same text as the title but in lowercase and hyphens instead of spaces.

The Template field is to choose the kind of element that should be created in this case the template should be Brand.

Once the brand folder has been created you can start creating the campaigns for it. Stand over the brand folder and click New on the right of the page.

The create page will appear with the following fields:

The Title field is to type in the title of the element to be created, in this case the name of the campaign.

The Name field is to type in the name of the campaign, is preferable to use the same text as the title but in lowercase and hyphens instead of spaces.

The Template field is to choose the kind of element that should be created in this case the template should be Campaign.

Once the campaign has been created you can start creating the campaign teasers. To do so stand over the campaign name on the right menu, then click on the list option from the right side of the page and then the New option.

The following window will appear:

The Title field is to type in the title of the element to be created, in this case the name of the teaser.

The Name field is to type in the name of the teaser, is preferable to use the same text as the title but in lowercase and hyphens instead of spaces.

The Template field is to choose the kind of element that should be created in this case the template should be Teaser Page.

After the teaser has been created the following configuration will appear:

Teaser Page tab:

The Title field is to change if desired the title of the teaser.

The Tags/Keywords field is to select the tags to identify the teaser as you do with pages (see Add a New Page).

The Description field is to type in a text describing the teaser.

        The Location field is to type in the path of the page where the teaser will be displayed.

Advance tab:

The On Time field is to select the date and time the teaser starts to be available.

The Off Time field is to select the date and time the teaser stops to be available.

The Segments field allows you to choose to whom the teaser will be direct to (age, gender, location, etc.).

Image tab:

This is the tab where the image for the teaser is uploaded. It’s recommended to upload files below 1MB and always upload them into the DAM, not directly.

Once the configuration has ended you can access the teaser (by double clicking on it’s name) to create it’s appearance by adding components to it as you do with pages. To see more about adding components go to Adding Components to a page.

The teaser is ready to be used. You can create more than one teaser for a campaign. To see how to add a teaser to a page go to  Teaser and see how the component works.

  1. Tagging

The CQ5 allows you to create tags for your site. The tags are names that you can use to connect and identify content on your pages that facilitate searches within the site and search engine optimization.

To add tags for your site click the last icon of the top menu of the System Admin UI.

This will take you to the Tagging section of the CQ5. To start creating tags stand over (click) the Universal Template folder on the left menu.

Then you can create a new group of tags for your site by clicking on the the New button presented at the right of the page.

The following window will appear:

The Title field is to type in the title for the tag. If the site is new is preferably to name the first tag after the site (ie. budweiser) and they add the desired tags within that one.

The Name field is to type in the name for the tag. This one is usually the same as the title but lower case and with hyphens instead of spaces giving that is the name that will appear in the urls.

The Description field is to type in a text explaining what is the tag created about.

Once created the tag you can choose to create subtags by standing over the new one (click) on the left menu and repeating the process just explained.

You can also choose to create the tags under the path the Universal Template >Tag. The tags saved under this path are standardized to appear in the search results as results filters.

To know more on how to add tags to a page see Edit Properties.

  1. Components Description

To add components/content to a page, you must open it in edition mode. From websites console you can go through the content tree to the page and double-click the icon to the left of the page name. Or you can select the site on the left panel, and then the page on the right panel of the console. Right-click the page and select Open in the menu displayed.

The selected page will open in a new browser tab. The first thing you will notice is the sidekick.

This toolbox contains all the components you can add to a page and it enables you to preview the changes, adjust page properties and even publish (activate) the page. By default it shows the edit mode, but has many features.

One thing you will notice is that every time you change something, the page will be reloaded showing your last modification. Therefore you do not need to save changes, just do them and then activate the page when you are ready to publish.

  1. Content Components

Most Used

Button

This component is used to add a button to your site, allowing the user to call a specific action (go to a specific page).

The URL field lets you type in/paste the URL where the user should land after clicking the button. The urls can be both from an external site (i.e http://www.example.com or http://example.com) or from within the platform (urls starting from the content level i.e /content/abinbev/en/home.html).

 

The Title field is to type in the button label.

The Size field is a dropdown to choose among small, default, large, extra small options By default, the second alternative is selected.

The External checkbox field should be checked to open the destination link in a new browser tab. Otherwise the link will be opened in the current tab.

The Show Modal checkbox field should be checked when you intend to open the content of the link in a modal window in the current browser tab. After reviewing the content user should close the modal window to return to the browser tab.

The Modal Id. field is to type in an Id to identify by code the instance of the modal window.

The Align field is a dropdown with three options: left, center, right; and lets you choose where to place the button within the page. Currently the value on this field has no effect on button position.

Embed Video

This component is used to embed a youtube video in your page. You should upload your video to a youtube channel - make it public so site visitors can play it - and then add its youtube id to the component.

Behavioral tab:

The field URL field is to type in/paste the video youtube id. This ID for youtube is the part of the url righ after the ‘=’ sign i.e the id of https://www.youtube.com/watch?v=PlLR9ANGsOo would be PlLR9ANGsOo.

Choose how to display the video has three options:

  • Modal window will display a modal with an x to close it on the upper right corner.
  • In site will show the video in the same place where the thumbnail is placed in the page.
  • External page will open a new browser tab for the user to watch the video.

Aesthetics tab:

The Video Title field is optional and allows you to enter a title for the video.

The Display title field if unchecked, the video title will not be displayed, even if the Video Title field is not empty.

The Display title and subtitle outside video frame, when checked, the title and subtitle will appear above the video thumbnail.

The Use thumbnail? should be used along with Use desktop and mobile image for hero video, Thumbnail and Mobile Thumbnail tab. If you add a video thumbnail but do not check Use thumbnail field, the site will display youtube default thumbnail for the video. If you want to have a specific thumbnail, check this field and add images in Thumbnail tab. If you want to have one thumbnail for desktop version and a different one for tablet/ipad and mobile versions, then check Use Thumbnail, Use desktop and mobile image for hero video and add images to Thumbnail and Mobile Thumbnail tabs.

The Use desktop and mobile image for hero video should be checked when there is one thumbnail for desktop version and a different one for tablet/ipad and mobile versions.

The Video subtitle is optional and lets you type in a subtitle for the video. Should be use together with Display subtitle checkbox. 

The Display subtitle, if unchecked, the video subtitle will not be displayed, even if the field is not empty.

The Share, if checked adds a share bar to the video so it can be shared in the social networks defined for the site.

The Disable suggested video should be checked when you do not want youtube suggesting/linking to other videos (that may be related to yours or not) when your video finishes playing.

The Alt text is a field to enter the text you would like to show when the user hovers over the video. This text is associated with an image that serves the same purpose and conveys the same essential information as the image. In situations where the image is not available to the reader, perhaps because they have turned off images in their web browser or are using a screen reader due to a visual impairment, the alternative text ensures that no information or functionality is lost. Absent or unhelpful alternative text can be a source of frustration for visually impaired users or site visitors.

Thumbnail tab:

In the Thumbnail tab you can add the image that will be used as video thumbnail in desktop and/or tablet/ipad devices. Maximum size allowed is full hd (1920 by 1080). This should be used together with Use Thumbnail?  and Use desktop and mobile image for hero video.

Mobile thumbnail tab:

In the Mobile thumbnail tab you can add the image that will be used as video thumbnail for mobile version. The aspect ratio should be 1:1 (square format) for better fit mobile screens. This should be used together with Use Thumbnail?  and Use desktop and mobile image for hero video.

Image

This component is used to insert an image into the page. This component can be configured in edit or in design modes.  For the design mode you need to have design rights.

The following configuration windows correspond to the edit mode.

Image Component Config tab:

The Title field is to enter the title of the image. It should be used together with Show Title?. 

The Show Title?, when checked will display the content of the Title field. When unchecked will not show a title even if Title field is not empty.

The Description is a field to type in some information about the image. 

The Description appears has three placement options to select: below the image only, on the modal (when clicked) or in both cases.

The Alt Text is a field to enter the text you would like to show when the user hovers over the image. This text is associated with an image that serves the same purpose and conveys the same essential information as the image. In situations where the image is not available to the reader, perhaps because they have turned off images in their web browser or are using a screen reader due to a visual impairment, the alternative text ensures that no information or functionality is lost. Absent or unhelpful alternative text can be a source of frustration for visually impaired users or site visitors.

The Style Option field allows you to choose the style in which the image are displayed.

The Img Attr Title field allows to input a text that will show when performing a hover over the image. It's used as a reference for those browsers that cannot display images.

The Optional Title Link field is to type in a url to where the Title will redirect you when clicking over it if desired.

The Image Link, is a field to input an url if is wanted for the image to serve as a link to another page.

The External is a field that should be checked to open the above link in a new browser tab. Otherwise the link will be open in the current tab.

The Show Modal Box? field let you choose whether or not to open a modal with the image once it is clicked.

Image tab:

This is the tab where the image is uploaded. This tab would contain the image that would be displayed on the desktop version of the page. It’s recommended to upload files below 1MB and always upload them into the DAM, not directly.

ImageMobile tab:

This is the tab where the image is uploaded. This tab would contain the image that would be displayed on the mobile version of the page. It’s recommended to upload files below 1MB and always upload them into the DAM, not directly.

Parallax settings tab:

The Active field tells the section where the image is contained to behave as parallax. This means that when the page is scrolled upwards the image will be displaced downwards and vice versa.  

The Speed ratio field indicates the speed on which the image will be displaced when scrolling the page.

The Background Speed Ratio field indicates the scrolling speed for the image background.

The Horizontal offset field indicates the horizontal offset of the parent to the edge of the screen

The Vertical offset field indicates the vertical offset of the parent to the edge of the screen

The Disable for Mobile field disables the parallax behavior for devices that meet specific resolutions.

Text

This component is used to add a text into your page. Usually is plain text but some format can be added. It is not recommended to add html within the text.

Text Configuration tab:

The text can be inserted with a title with regular font or italic using whether the Title or the Italic Title field respectively.

The Subtitle field allows you to input a subtitle to the text.

The Style Option field allows you to choose a style for the text.

Text tab:

The Text tab comes with a text editor window to introduce and format the text that will be displayed on the site.

Title

This component is used to insert a small text used only for titles or subtitles, but its default use is as page title.

The Select Title Type allows you to choose the kind of heading format the title is going to have.

The desired title is input on the Title field.

The Reference Name field is to type in an anchor tag with the name attribute set to the entered one.

The Add custom style allows you to choose an style for the title whiting a group of predefined styles.

        Other components

Bread Crumb

This component is used to add at the top of the page the path that the user has taken to be in the current page. The component does not need additional configuration after it has been added into the page.

 

Card List View Component

This component is used to retrieve the content from the event and blogs templates and show it in the page. It just displays the events or posts contained within a specific url.

 

The URL field is to type in the path of where the content templates are located.

The Max amount of pages field is to type in the number of pages/events to be displayed. By default it shows 10.

The Use Events Format checkbox is to indicate that the content type displayed are events.

The Enable infinite scroll checkbox is used when the number of events/pages to display are too many. An infinite scroll effect takes place when reaching the bottom of the page and then new results are retrieved.

The Infinite scroll offset field indicates the screen positioning in pixels where the load of new results will be triggered.  This field only works if the Enables infinite scroll was checked.

Children Pages Tree

This component is used to add a list of links showing the hierarchy of pages to which a page belongs to. The list will only show the pages from the 4rd (inclusive) level forward. Being the first level the site folder (i.e. budweiser), the second level the language folder (i.e En), and so on.

 

The component will show the all the pages from the 4th level even if they have checked the property Hide in Navigation. From the level 5th forward the component will only show the pages that don’t have checked said property.

Data Card

This component is used to insert a set of quantities or values into the page.

The Mode field let's you choose the way the information will be displayed. If the mode selected is Stat Mode, the card will display all fields except the Description field. If the mode chosen was Short Text Mode the card will only display the Title and the Description fields.

The Title field allows you to input a title for the data card.

The Description field allows you to enter a text regarding the information displayed.

Stat Mode Number, is where the value (number) is typed in.

Stat Mode Unit, is to indicate the unit used for measurements.

Stat Mode Details, is to describe or to add the name of what is being measured.

Download

This component is used to make available the download of a document or image to the user.

The Description field is to enter a text explaining what is the download about.

 

The File field is where the downloadable document will be held. Once the file has been dropped over the File section will show the name of the file. The name can be changed if desired without damaging the file and it will be the Title for the download displayed on the page.

 

 The Open file in new tab? checkbox lets you decide whether the file will open in a new or on the current tab.

Iframe

This component is used to insert content from another source, such as an advertisement, into a Web page.

The Type of target field allows you to choose the different options (Specific URL, Get parameter from URL window, and Get parameter with ID from window URL…) to target the content of the iframe.

The Parameter field is to type in the source of the iframe information only if you selected Get parameter from URL window in the Type of Target field. Do not complete this field otherwise.

The Domain field is to type in the domain from which the parameters should be taken only if you selected Get parameter from URL window in the Type of Target field. Do not complete this field otherwise.

The Width and Height fields are to input the dimensions of the iframe. This would make your iframe to have a fixed size.

The Same height as field allows you to choose the size of the iframe. The size then could be Fixed (see field above), Content (the iframe adjust to the content of the embed page), Window (the iframe adjust to the window dimensions), Inherit size from embed content (the iframe inherits its size from the embedded content).

The Show Scroll Bar? checkbox to choose whether or not to show the scroll bar on the iframe.

To create a New Site access to the Classic Siteadmin UI (see AEM Siteadmin brief introduction)  and stand over (click) the Websites folder on the left menu.

To update an existing image or document follow the next steps

  1. Go to the folder in the DAM where the current document is holded.
  2. Make sure that the new version of the file you want to update has the same name and extension of the old version of the file held in the DAM.
  3. Follow the same steps of Adding a new file (see above).
  4. The following message will appear:

To successfully replace the file you have to click the Replace button. After this the system will held only the new file and automatically replaced it in all the locations (pages) the file was previously used.
